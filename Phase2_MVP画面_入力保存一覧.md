# Phase 2：MVP画面（入力 -> 保存 -> 一覧）

## 目的
一番重要なユーザー導線を通す。

## やること
- 新規登録フォームを作る
- 一覧画面を作る
- 編集画面（最低限）を作る
- バリデーション（必須のみ）を入れる

## まとめるもの
- 画面ごとの「入力 -> 処理 -> 出力」メモ（短く）
- UIで困った点と改善案（TODO）

## 作業チェックリスト
- [ ] 新規登録フォームから保存できる
- [ ] 保存直後に一覧へ反映される
- [ ] 一覧から対象データを選んで編集できる
- [ ] 必須未入力時に保存できない
- [ ] エラーメッセージがユーザーに分かる文言になっている
- [ ] 画面ごとの入出力メモを残した

## 完了条件
- 「登録 -> 一覧 -> 編集」の導線が破綻せず、使っている感があること。
