# Phase 4：通知・外部連携（LINE / Slack 等）

## 目的
壊れやすい外部 I/O を後半フェーズに隔離して実装する。

## やること
- webhook 受け口を作る
- コマンド設計（例：今日、明日、一覧）を行う
- 通知テンプレート文を決める
- リトライと失敗時ログ（最低限）を実装する

## まとめるもの
- 連携の構成図（ざっくり）
- webhook 設定手順
- 送信失敗の観測方法（どこを見れば分かるか）

## 作業チェックリスト
- [ ] webhook の受信確認ができる
- [ ] 主要コマンドの入出力仕様を決めた
- [ ] 通知テンプレートを固定した
- [ ] タイムアウト / API エラー時のリトライ条件を定義した
- [ ] 失敗時ログの出力先と確認手順を決めた
- [ ] 手順書どおりに第三者が設定できる

## 完了条件
- 期待コマンドに対して期待返信が返ること。
- 失敗時でも原因を追えること。
