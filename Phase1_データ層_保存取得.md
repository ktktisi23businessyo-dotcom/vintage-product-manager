# Phase 1：データ層（保存と取得だけ先に作る）

## 目的
データが死なない土台を作る（UIは後でよい）。

## やること
- 保存先を確定する（DB / スプレッドシート / JSON など）。
- CRUD の最小 API（または関数）を作る。
- データモデル（必須項目）を確定する。

## まとめるもの
- データモデル（必須項目、型、例）
- 保存先の仕様（どこに何を保存しているか）
- サンプルデータと期待結果

## 作業チェックリスト
- [ ] 保存先の選定理由を1行で記載した
- [ ] `create`（1件追加）を実装した
- [ ] `list`（一覧取得）を実装した
- [ ] `update`（1件更新）を実装した
- [ ] `delete`（1件削除）を実装した
- [ ] 必須項目の未入力時にエラーとなることを確認した
- [ ] サンプルデータと期待結果を記録した

## 完了条件
- 1件追加 -> 一覧で取得 -> 更新、まで通ること。
